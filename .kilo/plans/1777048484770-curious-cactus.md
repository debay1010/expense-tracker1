# Change: add-kanban-board

## Proposal

### What
Add a kanban board feature to the expense-tracker application that allows users to organize expense entries visually using three columns: To-Do, Done, and Archived. The board will enable drag-and-drop functionality to move expenses between columns and reorder within columns.

### Why
The expense-tracker currently focuses on recording and visualizing expenses, but lacks a way to manage the workflow of expense processing. A kanban board will help users categorize expenses by their processing state:
- To-Do: Expenses that need attention or categorization
- Done: Processed/completed expenses
- Archived: Old or finalized expenses

This provides better organization and workflow management for expense tracking.

## Design

### Overview
The kanban board will be embedded as a new view within the existing expense tracker interface. It will display expense entries as cards that can be dragged between three columns representing different states.

### Technical Approach
- **Frontend**: Use vanilla JavaScript ES6+ with HTML/CSS for the UI components
- **Drag & Drop**: Integrate SortableJS library for smooth drag-and-drop functionality between columns and within columns
- **Data Storage**: Extend the existing SQLite schema (via sql.js) by adding a `status` column to the Expense table with values: 'todo', 'done', 'archived'
- **UI Integration**: Add the kanban board as a new tab/section in the main expense view, alongside existing expense list and chart views

### User Interface
- Three vertical columns: "To-Do", "Done", "Archived"
- Each expense displayed as a card showing key information (amount, date, category)
- Drag-and-drop enabled for moving cards between columns and reordering within columns
- Responsive design that works on different screen sizes

### Data Model Changes
- Add `status` VARCHAR column to Expense table with default value 'todo'
- Update existing expense creation to set appropriate initial status
- Add database migration to update existing expenses
